Feature set

Built around real job-seeker tasks.

The app is more than a form. It is a guided system for turning experience into stronger applications.

R

Resume builder

Structured sections, required fields, phone and address helpers, and PDF export.

A

ATS check

Compares the resume against a job description and shows capability gaps.

C

Cover letters

Creates tailored drafts from profile data, saved files, and the target job post.

I

Interview prep

Turns a role into likely questions and gives feedback on practice answers.

L

LinkedIn optimizer

Generates headline options, an About section, and a recruiter outreach message.

J

Job tracker

Save job postings and track application status from one place.

Architecture

Designed for a production path.

Flutter handles the app experience. Supabase handles auth, data, storage, and server-side AI functions. Provider keys stay on the backend, not inside the mobile app.

Flutter app Android now, stores later
Supabase Auth, database, storage, RLS
Claude API Planning and writing support
DeepSeek · Groq · Gemini Free AI fallback providers
GitHub Version control and Pages

Google sign in

Google authentication is used to create an account, identify the user, and save documents. The app requests basic profile scopes only.

Data ownership

Users keep ownership of their resumes, uploads, and generated documents. ApplyMate processes content only to provide requested features.
